## 1. Flywheel de Contenido

El **content flywheel** (volante de contenido) es un modelo que reemplaza el funnel tradicional, colocando al cliente en el centro de una estrategia circular y auto-sostenible.

### Las Tres Etapas Principales

| Etapa | Objetivo | Tipo de Contenido |
|-------|----------|-------------------|
| **Atraer** | Captar la atención de extraños | Posts educativos, guías de introducción, contenido SEO, redes sociales |
| **Involucrar** | Convertir visitantes en leads | Casos de estudio, comparativas, webinars, lead magnets |
| **Deleitar** | Convertir clientes en promotores | Tutoriales avanzados, contenido exclusivo, comunidad, soporte |

### Modelo de 4 Etapas (Expandido)

Algunos expertos añaden una cuarta etapa: **Analizar**. Esta fase se enfoca en métricas para entender qué contenido genera más engagement y duplicar esfuerzos en lo que funciona.

### Contenido por Etapa del Buyer Journey

#### Atracción (TOFU - Top of Funnel)
- Posts de blog informativos
- Infografías compartibles
- Videos cortos educativos
- Contenido en redes sociales
- **Intención de búsqueda**: Informacional

#### Consideración (MOFU - Middle of Funnel)
- Guías detalladas y ebooks
- Webinars y podcasts
- Comparativas de productos/servicios
- Casos de estudio
- **Intención de búsqueda**: Investigación/Comparación

#### Decisión (BOFU - Bottom of Funnel)
- Testimonios y reviews
- Demos y pruebas gratuitas
- Consultas personalizadas
- Ofertas especiales
- **Intención de búsqueda**: Transaccional

#### Deleite (Post-compra)
- Contenido exclusivo para clientes
- Tutoriales avanzados
- Programas de referidos
- Comunidad y foros

### Métricas Clave del Flywheel

- **Atracción**: Tráfico orgánico, nuevos visitantes, impresiones
- **Involucramiento**: Tasa de conversión leads, tiempo en página, páginas por sesión
- **Deleite**: NPS, tasa de retención, referencias generadas
- **General**: Costo de adquisición (CAC) - debe decrecer con el tiempo

## 2. Generación de Ideas de Contenido

### 2.1 Metodología de Topic Clusters (Pillar + Supporting Content)

El modelo de **pillar pages** y **topic clusters** es fundamental para SEO en 2025.

#### Estructura del Modelo

```
                    [Pillar Page]
                    (2,000-10,000 palabras)
                         |
        +----------------+----------------+
        |                |                |
   [Cluster 1]      [Cluster 2]      [Cluster 3]
   (800-1,500       (800-1,500       (800-1,500
    palabras)        palabras)        palabras)
```

#### Mejores Prácticas

- **Pillar Page**: Contenido extenso (mínimo 2,000 palabras) sobre un tema amplio
- **Cluster Content**: 8-12 artículos de soporte por cada pillar
- **Interlinking**: Link bidireccional entre pillar y clusters
- **Anchor text**: Descriptivo y consistente con el keyword principal

#### Beneficios Comprobados (2025)

- Contenido en clusters genera **30% más tráfico orgánico**
- Mantiene rankings **2.5x más tiempo** que posts independientes
- Mejora E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness)

### 2.2 Análisis de Preguntas Frecuentes

**Fuentes para encontrar preguntas:**
- Google "People Also Ask" (PAA)
- Answer The Public
- Quora y Reddit
- Comentarios en blogs de competidores
- FAQs de productos/servicios similares
- Consultas de clientes actuales

**Herramientas recomendadas:**
- Ahrefs (Questions report)
- SEMrush (Topic Research)
- AlsoAsked.com
- Google Search Console (consultas con interrogativos)

### 2.3 Evergreen vs Trending Content

| Aspecto | Evergreen | Trending |
|---------|-----------|----------|
| Duración | Años | 7-14 días |
| Ejemplo | "Guía de diseño sostenible" | "Tendencias Milán 2025" |
| Tráfico | Constante y creciente | Pico rápido, caída abrupta |
| Mantenimiento | Actualización anual | Obsoleto rápidamente |
| SEO | Alto valor a largo plazo | Bajo valor SEO sostenido |

**Ratio recomendado: 70-80% evergreen / 20-30% trending**

#### Estrategia Híbrida

1. Construir base sólida de contenido evergreen (guías, tutoriales, recursos)
2. Complementar con trending para engagement inmediato
3. Si trending funciona bien, convertirlo en evergreen (actualizar y expandir)

### 2.4 Calendario de Contenido Estacional

Para arquitectura de interiores (Jessica Méndez):

## 3. SEO On-Page para Blogs

### 3.1 Estructura de Títulos (Title Tags)

**Mejores prácticas:**
- **Longitud**: 50-60 caracteres (55 óptimo)
- **Keyword**: Al inicio del título cuando sea natural
- **Único**: Cada página debe tener título único
- **Legible**: Escribir para humanos, no solo buscadores

**Fórmulas efectivas:**
- [Número] + [Adjetivo] + [Keyword] + [Promesa]
- Cómo + [Acción] + [Beneficio]
- [Keyword]: Guía Completa [Año]

**Ejemplos:**
- "7 Principios del Diseño Sostenible para Interiores"
- "Cómo Elegir Materiales Eco-Friendly: Guía 2025"

### 3.2 Meta Descriptions

**Mejores prácticas:**
- **Longitud**: 150-160 caracteres
- **Keyword**: Incluir naturalmente (Google lo resalta en negritas)
- **CTA**: Usar lenguaje orientado a la acción
- **Único**: Una descripción diferente por página
- **Propuesta de valor**: Indicar qué ganará el lector

**Plantilla:**
```
[Beneficio principal]. [Qué aprenderás/encontrarás]. [CTA o elemento diferenciador].
```

**Ejemplo:**
```
Descubre cómo integrar materiales sostenibles en tu hogar. Guía práctica con
ejemplos reales de proyectos certificados. Incluye checklist descargable.
```

### 3.3 Uso de Keywords

**Ubicación estratégica:**
- Título (H1)
- Primer párrafo (primeras 100 palabras)
- Al menos un H2
- URL
- Meta description
- Alt text de imagen principal
- Naturalmente en el cuerpo (3-5 veces en artículo de 1,500 palabras)

**Densidad (2025):**
- No existe densidad "ideal" - enfocarse en naturalidad
- Evitar keyword stuffing
- Usar variaciones y sinónimos (LSI keywords)
- Priorizar intención de búsqueda sobre repetición

**Tipos de keywords por grupo:**
- 1 keyword principal por artículo
- 2-3 keywords secundarias relacionadas
- Keywords de cola larga (long-tail) para capturar intenciones específicas

### 3.4 Estructura de Headings

```
H1: Título principal (solo uno por página)
  └── H2: Sección principal 1
        └── H3: Subsección 1.1
        └── H3: Subsección 1.2
  └── H2: Sección principal 2
        └── H3: Subsección 2.1
              └── H4: Detalle (si necesario)
  └── H2: Conclusión / CTA
```

**Reglas:**
- Un solo H1 por página
- H2 para secciones principales
- No saltar niveles (H1 → H3)
- Keywords en headings cuando sea natural
- Headings deben funcionar como "escaneo" del artículo

### 3.5 Links Internos

**Cantidad recomendada:**
- 3-4 links internos por cada 1,500 palabras
- Más links en contenido más extenso
- Distribuir a lo largo del artículo

**Mejores prácticas:**
- Anchor text descriptivo (evitar "click aquí" o "leer más")
- Linkear desde contenido nuevo a contenido antiguo relevante
- Linkear desde contenido antiguo a contenido nuevo (revisar posts existentes)
- Evitar páginas huérfanas (sin links entrantes)
- Mantener profundidad máxima de 3 clicks desde homepage

**Estructura de topic clusters:**
- Pillar → todos los clusters
- Clusters → pillar + clusters relacionados

### 3.6 Links Externos

**Mejores prácticas:**
- Linkear a fuentes autoritativas y relevantes
- 2-3 links externos por artículo
- Abrir en nueva pestaña (`target="_blank"`)
- Verificar que links no estén rotos
- Evitar linkear a competencia directa

### 3.7 Optimización de Imágenes

#### Formato de Archivos

| Formato | Uso Recomendado |
|---------|-----------------|
| **WebP** | Formato preferido (26% menor que PNG, 25-34% menor que JPEG) |
| **AVIF** | Mejor compresión, soporte creciente |
| **JPEG** | Fotos, fallback para browsers antiguos |
| **PNG** | Imágenes con transparencia |
| **SVG** | Iconos, logos, gráficos vectoriales |

#### Nombres de Archivo

**Mal:** `IMG_1234.jpg`, `foto1.png`
**Bien:** `diseno-interior-sostenible-sala.webp`

**Reglas:**
- 5-7 palabras máximo
- Usar guiones (-), no guiones bajos (_)
- Sin caracteres especiales ni acentos
- Descriptivo del contenido
- Incluir keyword si es relevante

#### Alt Text

**Mejores prácticas:**
- Máximo 125 caracteres
- Describir el contenido de la imagen
- Incluir keyword principal si es natural
- NO usar "imagen de..." o "foto de..."
- Front-load keyword (al inicio cuando posible)

**Ejemplo:**
```
Mal: "imagen de sala de estar"
Bien: "Sala de estar minimalista con materiales reciclados y luz natural"
```

#### Compresión y Lazy Loading

- Comprimir imágenes antes de subir (TinyPNG, Squoosh)
- Implementar lazy loading para imágenes below the fold
- Especificar width y height para evitar CLS
- Incluir imágenes en sitemap XML

### 3.8 Schema Markup para Artículos

**Tipo de schema:** `BlogPosting` (para blogs) o `Article` (contenido general)

**Propiedades requeridas:**
```json
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "Título del artículo",
  "image": "URL de imagen destacada",
  "datePublished": "2025-01-15",
  "dateModified": "2025-01-15",
  "author": {
    "@type": "Person",
    "name": "Jessica Méndez",
    "url": "https://jessicamendez.bio/sobre-mi"
  },
  "publisher": {
    "@type": "Organization",
    "name": "Jessica Méndez",
    "logo": {
      "@type": "ImageObject",
      "url": "https://jessicamendez.bio/logo.png"
    }
  },
  "description": "Meta description del artículo"
}
```

**Schemas adicionales útiles:**
- **HowTo**: Para tutoriales paso a paso
- **FAQ**: Para secciones de preguntas frecuentes
- **BreadcrumbList**: Para navegación
- **VideoObject**: Si el post incluye video

**Validación:** Usar [Rich Results Test](https://search.google.com/test/rich-results) de Google

## 4. Estructura Ideal de un Post

### 4.1 Anatomía del Post Perfecto

```
┌─────────────────────────────────────────┐
│  TÍTULO (H1)                            │
│  50-60 caracteres, keyword al inicio    │
├─────────────────────────────────────────┤
│  IMAGEN DESTACADA                       │
│  WebP, alt text optimizado              │
├─────────────────────────────────────────┤
│  INTRODUCCIÓN                           │
│  1-3 párrafos, hook + contexto          │
│  Keyword en primeras 100 palabras       │
├─────────────────────────────────────────┤
│  TABLA DE CONTENIDOS (opcional)         │
│  Para posts >2000 palabras              │
├─────────────────────────────────────────┤
│  CUERPO                                 │
│  H2 > H3 > H4 jerárquicos               │
│  Párrafos cortos (2-3 oraciones)        │
│  Listas, imágenes, ejemplos             │
│  Links internos y externos              │
├─────────────────────────────────────────┤
│  CONCLUSIÓN                             │
│  Resumen de puntos clave                │
│  Respuesta a la promesa del título      │
├─────────────────────────────────────────┤
│  CTA                                    │
│  Próximo paso claro y natural           │
├─────────────────────────────────────────┤
│  AUTOR / BIO (opcional)                 │
│  Refuerza E-E-A-T                       │
└─────────────────────────────────────────┘
```

### 4.2 La Introducción que Engancha

**Estructura PAS (Problem-Agitate-Solve):**
1. **Problem**: Identificar el problema/dolor del lector
2. **Agitate**: Amplificar las consecuencias
3. **Solve**: Presentar la solución (tu artículo)

**Estructura APP (Agree-Promise-Preview):**
1. **Agree**: Validar al lector ("Sé que...")
2. **Promise**: Qué logrará al leer
3. **Preview**: Adelanto del contenido

**Longitud:** 1-3 párrafos (50-150 palabras)

**Ejemplo (PAS):**
> "Diseñar un espacio sostenible puede parecer abrumador con tantas opciones de materiales y certificaciones. Sin una guía clara, es fácil terminar gastando de más en soluciones que no son tan eco-friendly como prometen. En este artículo, te comparto los 7 principios fundamentales que aplico en cada proyecto para crear interiores verdaderamente sostenibles sin complicaciones."

### 4.3 Longitud Recomendada por Tipo de Contenido

| Tipo de Contenido | Palabras | Uso |
|-------------------|----------|-----|
| Post corto | 300-600 | Noticias, actualizaciones |
| Post estándar | 800-1,500 | Posts regulares, tutoriales simples |
| Post largo | 1,500-2,500 | Guías, how-to detallados |
| **Sweet spot SEO** | **1,900-2,450** | Contenido evergreen |
| Pillar page | 2,000-10,000+ | Guías definitivas, recursos completos |

**Regla de oro:** La longitud depende de la intención de búsqueda y la competencia. Analizar qué rankea en la primera página para tu keyword.

### 4.4 Elementos de Formato

**Para mejorar legibilidad:**
- Párrafos de 2-3 oraciones máximo
- Listas con viñetas o números
- Destacados (bold, callouts)
- Imágenes cada 300-500 palabras
- Espaciado generoso
- Subtítulos descriptivos

**Elementos interactivos:**
- Tablas comparativas
- Infografías
- Videos embebidos
- Calculadoras/herramientas
- Quotes destacados

### 4.5 CTAs Efectivos

**Tipos de CTA según objetivo:**

| Objetivo | CTA Ejemplo |
|----------|-------------|
| Newsletter | "Suscríbete para recibir tips semanales" |
| Lead magnet | "Descarga la checklist gratuita" |
| Servicios | "Agenda una consulta sin costo" |
| Contenido relacionado | "Lee también: [Artículo relacionado]" |
| Comunidad | "Únete a nuestra comunidad" |
| Comentarios | "¿Qué opinas? Cuéntamelo en comentarios" |

**Mejores prácticas:**
- CTA principal al final del post
- CTAs secundarios en puntos naturales (no intrusivos)
- Lenguaje en primera persona ("Quiero mi guía" vs "Descargar guía")
- Debe sentirse como paso natural, no pitch de ventas
- Máximo 2-3 CTAs diferentes por post

## 5. Taxonomía: Categorías y Tags

### 5.1 Diferencias Fundamentales

| Aspecto | Categorías | Tags |
|---------|------------|------|
| Propósito | Agrupar por tema amplio | Describir detalles específicos |
| Estructura | Jerárquica (padres/hijos) | Plana (sin jerarquía) |
| Cantidad | 5-10 en total | Ilimitadas (pero controladas) |
| Por post | 1-2 categorías | 5-10 tags |
| Ejemplo | "Diseño Sostenible" | "madera-reciclada", "certificación-leed" |
| Analogía | Capítulos de un libro | Índice temático |

### 5.2 Mejores Prácticas para Categorías

**Cantidad recomendada:** 5-10 categorías totales (7±2 como regla)

**Para jessicamendez.bio (ejemplo):**
```
Categorías propuestas:
├── arquitectura        (proyectos y técnicas de arquitectura)
├── diseno-sostenible   (materiales eco, certificaciones, prácticas)
├── inspiracion         (referencias, tendencias, mood boards)
├── proyectos           (casos de estudio, portfoliio)
└── proceso-creativo    (behind the scenes, metodología)
```

**Reglas:**
- Cada post en 1-2 categorías máximo
- No duplicar con tags (si existe categoría "sostenibilidad", no crear tag "sostenibilidad")
- Nomenclatura consistente (singular o plural, no mezclar)
- URL amigables: `/blog/diseno-sostenible/` no `/blog/Diseño%20Sostenible/`

### 5.3 Mejores Prácticas para Tags

**Cantidad por post:** 5-10 tags (máximo 15 combinando categorías)

**Ejemplos de tags útiles:**
- Materiales: `madera`, `bambú`, `corcho`, `textiles-organicos`
- Estilos: `minimalista`, `escandinavo`, `wabi-sabi`
- Espacios: `sala`, `cocina`, `oficina-en-casa`
- Conceptos: `iluminacion-natural`, `biofilia`, `economia-circular`

**Reglas:**
- Usar minúsculas y guiones: `diseño-biofílico` no `Diseño Biofílico`
- Singular o plural consistente (elegir uno)
- Mínimo 2-3 posts por tag (si solo aplica a uno, no crear tag)
- Revisar y consolidar tags periódicamente

### 5.4 SEO para Taxonomías

**Indexación:**
- Considerar no indexar páginas de archivo de tags (pueden crear contenido thin)
- Indexar páginas de categorías principales
- Agregar descripción única a cada página de categoría

**Optimización:**
- Añadir intro con keyword a páginas de archivo de categorías
- Usar canonical tags para evitar duplicados
- Auditar taxonomías trimestralmente

### 5.5 Naming Conventions

**Formato recomendado:**
```
# URLs
minusculas-con-guiones

# Ejemplos correctos
diseno-sostenible
materiales-eco
inspiracion-2025

# Ejemplos incorrectos
Diseño_Sostenible
materiales eco
INSPIRACIÓN
```

**Evitar:**
- Caracteres especiales (ñ → n, á → a)
- Espacios (usar guiones)
- Underscores
- Mayúsculas
- Números solos como categoría
- Nombres genéricos ("general", "otros", "misc")

## Checklist de Implementación

### Antes de Escribir
- [ ] Keyword research completo
- [ ] Definir cluster (pillar o supporting)
- [ ] Analizar competencia en SERP
- [ ] Definir intención de búsqueda
- [ ] Crear outline con H2/H3

### Durante la Escritura
- [ ] H1 único con keyword
- [ ] Keyword en primeras 100 palabras
- [ ] Estructura jerárquica de headings
- [ ] Párrafos de 2-3 oraciones
- [ ] 3-4 links internos por 1,500 palabras
- [ ] 2-3 links externos a fuentes autoritativas
- [ ] Imágenes optimizadas (WebP, alt text)
- [ ] CTA claro y natural

### Antes de Publicar
- [ ] Meta title (50-60 caracteres)
- [ ] Meta description (150-160 caracteres)
- [ ] URL amigable con keyword
- [ ] Imagen destacada optimizada
- [ ] Schema markup implementado
- [ ] 1-2 categorías asignadas
- [ ] 5-10 tags relevantes
- [ ] Preview en móvil

### Después de Publicar
- [ ] Compartir en redes sociales
- [ ] Agregar links internos desde posts antiguos
- [ ] Verificar indexación (Search Console)
- [ ] Monitorear métricas primera semana
